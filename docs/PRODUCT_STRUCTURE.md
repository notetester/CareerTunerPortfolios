# CareerTuner Product Structure

이 문서는 CareerTuner를 사용자 관점에서 어떻게 이해할지 정리한 공통 컨텍스트다.
기획 원본은 `docs/planning/기획.txt`이고, 이 문서는 실제 메뉴와 기능 구조를 개발자가 빠르게 공유하기 위한 지도다.

핵심 판단은 두 가지다.

- 제품의 중심 단위는 공고가 아니라 **지원 건(Application Case)** 이다.
- 화면 메뉴 구조와 내부 개발 모듈 구조는 비슷하지만 완전히 같지는 않다.

## 1. 공개 영역

```text
공개 영역
 ├─ 홈
 │   ├─ 서비스 핵심 소개
 │   ├─ 공고 업로드 기반 AI 분석 소개
 │   ├─ 지원 건 관리 흐름 소개
 │   ├─ AI 면접 기능 소개
 │   └─ 시작하기 / 로그인 유도
 │
 ├─ 서비스 소개
 │   ├─ CareerTuner란?
 │   ├─ 기존 취업 사이트와의 차이
 │   ├─ 공고 분석 기능 소개
 │   ├─ 스펙 비교 기능 소개
 │   ├─ AI 면접 기능 소개
 │   └─ 장기 취업 경향 분석 소개
 │
 ├─ 요금제
 │   ├─ 무료/베이직/프로/프리미엄 플랜
 │   ├─ 크레딧 충전
 │   └─ 기능별 크레딧 차감 안내
 │
 ├─ 회사/브랜드 소개
 │   ├─ 회사 소개
 │   ├─ 팀 소개
 │   ├─ 채용
 │   ├─ 블로그
 │   ├─ 보도자료
 │   └─ 공식 채널
 │
 ├─ 고객센터
 │   ├─ 사용 가이드
 │   ├─ FAQ
 │   ├─ 공지사항
 │   └─ 문의하기
 │
 └─ 법적 문서
     ├─ 이용약관
     ├─ 개인정보처리방침
     ├─ AI 데이터 이용 동의
     └─ 저작권/공고문 이용 안내
```

공개 영역은 회원 전환과 신뢰 확보가 목적이다. 기능 구현이 늦어져도 라우트와 페이지의 위치는 유지한다.

## 2. 인증/회원 영역

```text
인증/회원
 ├─ 로그인
 ├─ 회원가입
 ├─ 이메일 인증
 ├─ 비밀번호 찾기
 ├─ 소셜 로그인
 ├─ 토큰 재발급
 └─ 로그아웃
```

현재 이메일/JWT 기반 인증이 먼저 구현되어 있다. 카카오/네이버/구글 OAuth와 이메일 발송 API 키는 나중에 실제 값으로 교체한다.

## 3. 사용자 메인 영역

```text
사용자 메인
 ├─ 대시보드
 ├─ 내 프로필
 ├─ 지원 건 관리
 ├─ AI 가상 면접
 ├─ AI 첨삭
 ├─ 취업 분석
 ├─ 커뮤니티
 ├─ 결제/구독
 ├─ 알림
 └─ 설정
```

### 대시보드

```text
대시보드
 ├─ 전체 준비 현황
 ├─ 활성 지원 건 요약
 ├─ 최근 지원 건
 ├─ 최근 AI 분석 결과
 ├─ 최근 모의면접 결과
 ├─ 부족 역량 요약
 ├─ 오늘의 할 일
 ├─ 보유 크레딧
 └─ 최근 알림
```

대시보드는 독립 데이터 소유자가 아니라 여러 도메인을 모아 보여주는 요약 화면이다.
참조 도메인은 `applicationcase`, `fitanalysis`, `interview`, `payment`, `ai_usage_log`, `notification`이다.

### 내 프로필

```text
내 프로필
 ├─ 기본 정보
 ├─ 이력서 관리
 ├─ 자기소개서 관리
 ├─ 경력/프로젝트 관리
 ├─ 기술스택 관리
 ├─ 자격증/학력/어학 관리
 └─ 포트폴리오 관리
```

프로필은 공고 분석과 적합도 분석의 비교 기준이다. 이력서/포트폴리오 파일은 `file` 도메인과 연결될 수 있다.

### 지원 건 관리

```text
지원 건 관리
 ├─ 지원 건 목록
 │   ├─ 전체/진행 중/완료/보관
 │   ├─ 즐겨찾기
 │   ├─ 검색
 │   ├─ 필터
 │   └─ 정렬
 │
 ├─ 새 지원 건 만들기
 │   ├─ 기업명
 │   ├─ 직무명
 │   ├─ 공고일
 │   ├─ 공고 출처
 │   └─ 공고문 등록 방식
 │
 ├─ 공고문 관리
 │   ├─ 텍스트 붙여넣기
 │   ├─ PDF 업로드
 │   ├─ 이미지 업로드
 │   ├─ URL 입력
 │   ├─ 수동 입력
 │   └─ 공고문 원문 보기
 │
 └─ 지원 건 상세
     ├─ 개요
     ├─ 공고 분석
     ├─ 기업 분석
     ├─ 내 스펙과 비교
     ├─ 지원 전략
     ├─ 학습/자격증 추천
     ├─ 예상 면접 질문
     ├─ 가상 면접 연결
     ├─ 첨삭 기록
     └─ 지원 건 리포트
```

지원 건 상세는 CareerTuner의 핵심 작업공간이다. 한 기업/공고/직무 조합에 대한 분석, 전략, 면접, 첨삭 기록이 이 안에 모인다.

### AI 가상 면접

```text
AI 가상 면접
 ├─ 면접 시작
 │   ├─ 지원 건 선택
 │   ├─ 면접 모드 선택
 │   ├─ 질문 수 / 시간 제한 선택
 │   └─ 음성/영상 사용 여부 선택
 │
 ├─ 면접 모드
 │   ├─ 기본 면접
 │   ├─ 직무/기술 면접
 │   ├─ 인성/압박 면접
 │   ├─ 자기소개서 기반 면접
 │   ├─ 포트폴리오 기반 면접
 │   └─ 실전 종합 면접
 │
 ├─ 면접 진행
 ├─ 답변 평가
 ├─ 음성/태도 분석
 └─ 면접 리포트
```

표정, 시선, 자세 분석은 합격/불합격 판단이 아니라 면접 태도 개선 참고자료로만 다룬다.

### AI 첨삭

```text
AI 첨삭
 ├─ 면접 답변 첨삭
 ├─ 자기소개서 첨삭
 ├─ 이력서 첨삭
 └─ 포트폴리오 설명 첨삭
```

공통 첨삭은 `/correction`에서 제공하고, 지원 건 기반 첨삭 기록은 `/applications/:id`와 연결한다.

### 취업 분석

```text
취업 분석
 ├─ 전체 지원 경향
 ├─ 역량 분석
 ├─ 직무별 준비도
 ├─ 면접 점수 변화
 └─ 추천 지원 방향
```

취업 분석은 여러 지원 건을 종합하는 기능이다. 단일 지원 건 상세와 구분한다.

### 커뮤니티

```text
커뮤니티
 ├─ 취업 후기
 ├─ 면접 후기
 ├─ 직무별 질문 공유
 ├─ 합격 전략 게시판
 ├─ 포트폴리오 피드백
 ├─ 자격증 후기
 ├─ 자유게시판
 ├─ 댓글
 ├─ 좋아요/북마크
 └─ 신고
```

면접 후기는 AI 요약과 질문 추출이 자연스럽게 붙을 수 있는 영역이다.

### 결제/구독

```text
결제/구독
 ├─ 요금제
 ├─ 크레딧
 ├─ AI 사용량
 ├─ 결제 내역
 └─ 구독 관리
```

`users.credit`는 현재 잔액이고, 장기적으로는 `credit_transaction` 장부가 필요하다.

### 알림

```text
알림
 ├─ 분석 완료 알림
 ├─ 면접 리포트 완료 알림
 ├─ 크레딧 부족 알림
 ├─ 결제 알림
 ├─ 공지 알림
 ├─ 문의 답변 알림
 └─ 읽음 처리
```

헤더와 설정 화면에 이미 알림 UI가 있으므로 `notification`은 실제 기능 폴더로 유지한다.

### 설정

```text
설정
 ├─ 계정 설정
 ├─ 개인정보 관리
 ├─ AI 데이터 사용 동의
 └─ 알림 설정
```

동의 이력은 `consent` 도메인에서 버전별로 남긴다.

## 4. 관리자 영역

```text
관리자
 ├─ 관리자 대시보드
 ├─ 회원 관리
 ├─ 결제 관리
 ├─ AI 사용량 관리
 ├─ 게시판/신고 관리
 ├─ 공지사항 관리
 ├─ FAQ/고객문의 관리
 ├─ 요금제 관리
 ├─ 프롬프트 템플릿 관리
 └─ 시스템 로그
```

관리자 프런트는 별도 앱이 아니라 `frontend/src/admin` 안에서 운영한다.
관리자 기능별 화면은 `frontend/src/admin/features/<feature>`에 둔다.

## 5. 내부 시스템 기능

```text
내부 시스템
 ├─ AI 파이프라인
 │   ├─ 공고문 분석 AI
 │   ├─ 기업 분석 AI
 │   ├─ 스펙 비교 AI
 │   ├─ 학습/자격증 추천 AI
 │   ├─ 면접 질문 생성 AI
 │   ├─ 답변 평가 AI
 │   ├─ 첨삭 AI
 │   ├─ 장기 취업 경향 분석 AI
 │   ├─ 프롬프트 템플릿
 │   └─ AI 사용량 기록
 │
 ├─ 파일/스토리지
 ├─ 알림 시스템
 ├─ 크레딧 장부
 ├─ 동의/약관 이력
 └─ 배포/운영
```

내부 시스템 기능은 상단 메뉴에 그대로 노출되지 않아도 도메인 폴더는 필요하다.
