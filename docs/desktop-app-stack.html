먼저 용어부터

낯선 단어 풀이

C++ (씨쁠쁠)
프로그램을 만드는 빠르고 강력한 언어. 크롬 브라우저, 게임 엔진(언리얼), 윈도우 프로그램 상당수가 C++로 만들어진다.
Qt (큐트)
C++로 데스크탑 앱을 만들게 해주는 도구 세트(프레임워크). 화면 그리기·네트워크·파일 처리가 다 들어있다. 한 번 만들면 윈도우·맥·리눅스에서 전부 돈다.
QML (큐엠엘)
Qt에서 화면(UI)을 만드는 전용 언어. HTML+CSS+JavaScript와 느낌이 비슷해서 웹 해본 사람은 금방 배운다. 애니메이션이 강점.
CMake (씨메이크)
흩어진 소스코드들을 실행파일로 묶는(빌드) 레시피를 적는 도구. "이 파일들을 이렇게 컴파일해" 라고 알려준다.
컴파일
사람이 쓴 코드(.cpp)를 컴퓨터가 바로 실행하는 파일(.exe)로 번역하는 과정.
네이티브 앱
브라우저(웹뷰)로 감싼 게 아니라 OS에서 직접 도는 진짜 프로그램. 우리가 목표한 "제대로 된 하나의 프로그램".
SSE
서버가 클라이언트로 진행 상황을 실시간으로 흘려보내는 기술. "분석 끝남 → 질문 생성됨 → ..."이 실시간으로 뜨는 이유.
왜 이걸로?

C++ / Qt 를 고른 이유

🧱

진짜 네이티브

웹을 감싼 게 아니라 OS에서 직접 도는 무거운 프로그램. 우리가 처음에 정한 "제대로 된 하나의 프로그램" 목표에 정확히 맞는다.

🌍

크로스플랫폼

한 번 짜면 윈도우·맥·리눅스 전부에서 돈다. 코드를 OS별로 다시 안 짜도 된다. (윈도우 전용 도구로는 못 하는 것)

✨

모던 UI 가능

QML로 우리가 만든 프로토타입의 애니메이션·실시간 진행·토스트를 그대로 구현한다. 화면이 부드럽고 살아있다.

핵심: 데스크탑 앱이 하는 일은 "화면을 그리고 + 서버와 통신"이다. 무거운 AI 계산은 서버가 한다(이전에 정한 결정). C++/Qt는 이 네이티브 클라이언트를 가장 단단하게, 그리고 여러 OS에서 한 번에 만들 수 있게 해준다.
앱의 속 구조

C++ 는 "엔진", QML 은 "얼굴"

앱은 두 부분으로 나뉜다. 안 보이지만 일하는 C++ 코어(엔진)와, 눈에 보이는 QML 화면(얼굴). 자동차로 치면 엔진과 계기판이다.

🎨 QML — 얼굴 (화면)
사용자가 보고 누르는 부분. 버튼·작업 카드·진행률 바·애니메이션·토스트.
대시보드작업 상세면접 연습리포트폰 미러
↕ 서로 신호를 주고받음 (C++가 데이터를 주면 → QML이 그림)
⚙️ C++ — 엔진 (두뇌)
안 보이지만 실제로 일하는 곳. 서버와 통신하고, 데이터를 관리하고, 로그인 토큰을 다룬다.
ApiClient (서버 호출)SseClient (실시간)AuthService (로그인)JobModel (데이터)
↕ 인터넷 (HTTP / SSE)
🖥️ 서버 (기존 Spring + 4090)
실제 AI 작업을 돌리는 곳. 데스크탑 앱은 여기에 "접속"만 한다. 새로 만들지 않고 기존 것을 그대로 쓴다.
왜 나누나? 화면 담당(QML)과 일 담당(C++)을 분리하면, 화면 디자인을 바꿔도 엔진은 안 건드리고, 반대로 서버 통신 로직을 고쳐도 화면은 그대로다. 유지보수가 쉬워진다.
프로젝트 구조

파일이 이렇게 생긴다

CareerTuner.Desktop/ # C++17 · Qt 6 데스크탑 앱 (별도 프로젝트) ├─ CMakeLists.txt # 빌드 레시피 (어떤 파일을 어떻게 묶을지) ├─ main.cpp # 앱 시작점 · 트레이(시계 옆 아이콘) 상주 ├─ qml/ # ── 얼굴(화면) ── │ ├─ Main.qml # 전체 셸 (왼쪽 사이드바 + 화면 전환) │ ├─ DashboardPage.qml # 작업 큐 카드 목록 │ ├─ JobDetailPage.qml # 의존그래프 + 단계별 결과 │ ├─ PracticePage.qml # 면접 연습 (질문·답변·녹음) │ ├─ ReportPage.qml # 면접 리포트 (점수·평가) │ └─ PhoneMirror.qml # 우측 폰 미러 (모바일 동기화) ├─ core/ # ── 엔진(로직) ── │ ├─ ApiClient # 서버 REST 호출 + 로그인 토큰(JWT) 첨부 │ ├─ SseClient # 실시간 스트림 받기 + 끊기면 자동 재연결 │ ├─ AuthService # 로그인 · 토큰 갱신 │ └─ JobModel # 작업 목록 데이터 (화면에 자동 연결) └─ resources/ # 아이콘 · 다크 테마 스타일
협업 안전: 이 데스크탑 앱은 완전히 별도 프로젝트다. 기존 backend/·frontend/ 코드를 건드리지 않으므로 다른 팀원 작업과 충돌하지 않는다.
핵심 기능 하나

"끊겨도 안 놓친다" — 실시간 진행 받기

면접 준비 작업은 시간이 걸린다. 서버가 진행 상황을 실시간으로 찔러주면(SSE), C++의 SseClient가 받아서 화면(QML)에 바로 표시한다.

문제는 인터넷이 끊기거나 노트북을 닫았을 때다. 보통이면 그 사이 진행을 놓친다. 우리는 이렇게 푼다:

① 받은 이벤트마다 번호(seq)가 붙는다. ② 앱은 "마지막으로 본 번호"를 기억한다. ③ 다시 연결될 때 "나 N번까지 봤어"라고 서버에 알린다(Last-Event-ID). ④ 서버는 N번 다음부터 다시 보내준다.

비유: 드라마를 보다 잠들어도, 넷플릭스가 "여기까지 봤어요" 기억했다가 그 다음 화부터 틀어주는 것과 같다. C++로 이 동작을 충분히 깔끔하게 만들 수 있다.
헛일 아님

만든 프로토타입이 그대로 설계도가 된다

앞서 만든 클릭 가능한 프로토타입(HTML)의 화면들이 Qt(QML) 컴포넌트로 1:1로 옮겨진다. 디자인을 새로 짜는 게 아니다.

좌측 사이드바Main.qml 의 네비화면 전환 메뉴

만드는 순서

데스크탑 앱 구현 단계

STEP 1 · 골격

앱이 켜지고 서버에 붙는다

  • 메인 창 + 사이드바 (Main.qml)
  • 로그인 → JWT 토큰 받기
  • 대시보드 + 작업 카드
  • SseClient — 실시간 진행 받기
  • 트레이 상주
STEP 2 · 화면

주요 화면을 채운다

  • 작업 상세 (의존그래프)
  • 면접 연습 화면
  • 면접 리포트
  • 완료 알림(Windows 토스트)
STEP 3 · 네이티브

데스크탑만의 강점

  • 고화질 녹화 (화면 + 웹캠)
  • 폰 미러 · 디스패치 연동
  • 크로스플랫폼 빌드
