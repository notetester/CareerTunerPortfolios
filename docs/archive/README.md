# 보관 문서 안내

이 폴더는 역사적 맥락은 가치가 있지만 현재 정본으로 사용하면 안 되는 문서를 연월별로 보관한다.
완료된 인계, 특정 시점의 감사, 병합 판단, 후속 구현으로 대체된 설계 초안이 대상이다.

## 보관 기준

다음 조건 중 하나를 만족하면 보관 후보로 분류한다.

- 수행이 끝난 일회성 작업 지시나 인계
- 기준 SHA·날짜가 고정된 감사 결과로서 현재 상태를 계속 대표하지 않는 문서
- 후속 결정이나 구현으로 대체된 설계 초안
- 재현 가능한 정본에 흡수된 진행 로그

아직 유효한 정책, 반복 실행하는 runbook, 현재 검증 원장은 보관하지 않는다.

## 이동 원칙

- 원래 작성일과 당시 기준 commit/PR을 보존한다.
- 문서 상단에 `상태: archived`, 보관 사유, 대체 정본을 적는다.
- 사실을 현재 시제로 다시 쓰지 않고 당시 판단임을 명확히 한다.
- 파일명은 소문자 ASCII `kebab-case`를 사용한다.
- 이동과 모든 추적 참조 수정을 같은 커밋에 수행한다.
- 외부 참조 가능성이 있으면 구경로에 짧은 안내를 한 릴리스 동안 유지한다.

보관은 삭제를 미루는 장소가 아니다. 개인정보·비밀값·불필요한 raw output은 archive에도 남기지 않는다.
