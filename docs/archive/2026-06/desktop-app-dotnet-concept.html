CareerTuner 데스크탑 앱 구상
— 면접 준비 컨트롤 센터

웹을 그냥 감싼 데스크탑이 아니라, 작업을 돌리고 기기 사이로 던지는 "관제탑" 프로그램을 만든다. 이 문서는 그 그림을 그림과 화면 목업으로 설명한다.

한 줄 요약

데스크탑은 "코덱스/클로드코드의 면접판"이다

지금 우리 제품은 웹 + 모바일 앱이 있다. 여기에 윈도우 데스크탑 앱을 추가하는데, 핵심 원칙은 하나다.

👉 웹/모바일로는 못 하거나 불편한 일만 데스크탑이 맡는다.

데스크탑의 역할은 "면접 준비 작업을 돌리고 관리하는 관제탑"이다. 여러 면접 준비 작업을 동시에 돌리고, 진행 상황을 한눈에 보고, 그 작업을 폰으로 던지거나 폰에서 이어받는다. 개발자가 쓰는 Claude Code / Codex 데스크탑 클라이언트를 떠올리면 된다 — 단지 대상이 "코드"가 아니라 "면접 준비"일 뿐이다.

먼저 용어 정리

이 문서에 나오는 단어

작업 (Job)
"삼성전자 SW직군 면접 준비해줘" 같은 한 덩어리 일거리. 안에 여러 단계(공고분석 → 예상질문 → 모의면접 → 평가 → 리포트)가 들어있고, 시간이 좀 걸린다.
러너 (Runner)
그 작업을 실제로 실행하는 주체. 우리는 이걸 서버로 정했다. (밑에서 설명)
오케스트레이터
작업의 단계들을 순서·의존관계에 맞춰 자동으로 돌려주는 우리 엔진. 이미 만들어 둔 게 있다(코드명 autoprep).
SSE
서버가 클라이언트로 진행 상황을 실시간으로 흘려보내는 기술. "분석 끝남 → 질문 5개 생성됨 → ..." 이 실시간으로 화면에 뜨는 이유.
디스패치
작업을 기기 사이로 보내고 이어받는 것. 데스크탑에서 시작한 걸 폰에서 확인하거나, 폰에서 시작해 서버가 돌리거나.
네이티브 앱
웹뷰(브라우저)로 감싼 게 아니라, 그 OS 전용 언어로 만든 진짜 프로그램. 데스크탑은 C#/.NET으로 만든다.
전체 구조

누가 일을 하고, 누가 보여주나

제일 중요한 결정: 작업을 실제로 실행하는 "러너"는 서버다. 데스크탑·모바일·웹은 모두 그 작업을 보고·조종하는 클라이언트일 뿐이다.

"작업 하나"가 도는 과정

1. 사용자가 아무 기기(데스크탑/폰/웹)에서 "이 공고로 면접 준비" 요청 → 서버에 작업(Job)이 하나 생기고 고유 번호를 받는다.

2. 서버(러너)가 오케스트레이터로 단계들을 자동 실행한다. 진행 상황을 DB에 저장하면서 SSE로 실시간 송출한다.

3. 데스크탑·폰은 그 작업 번호로 접속해 진행을 본다. 접속을 끊었다 다시 붙어도 저장돼 있으니 이어서 보인다. → 이게 "이어받기"의 원리.

4. 작업이 끝나면 서버가 푸시 알림을 보낸다. 폰·데스크탑 어디서든 결과(리포트)를 연다.

핵심: "작업 번호로 어디서든 다시 붙는다"는 한 가지 설계가 ①알림 ②트리거 ③이어받기를 전부 가능하게 한다. 그래서 이걸 가장 먼저 만든다.
만드는 순서

한 번에 다 말고, 3단계로

STEP 1 · MVP

관제탑 골격 + 알림

  • 서버에 작업(Job) 저장 구조
  • 데스크탑 관제탑 화면(작업 큐·실시간 로그·트레이)
  • 작업 완료 푸시 알림
  • → "코덱스 데스크탑" 느낌 완성, 데모 임팩트 즉발
STEP 2

기기 간 디스패치

  • 폰에서 작업 트리거
  • 세션 이어받기(PC ↔ 폰)
  • 기기 페어링/온라인 표시
  • → STEP1을 잘 깔았으면 거의 따라온다
STEP 3

네이티브 캡처

  • 데스크탑 고화질 녹화(화면+웹캠)
  • 폰을 면접 입력장치로
  • 캡처 → 서버 분석 파이프라인
  • → 가장 무거움, 마지막

기술 스택

데스크탑 C#/.NET 네이티브 (WPF 또는 WinUI 3)

백엔드 기존 Spring REST(/api/**) 그대로 사용. 재구현 안 함. C#은 HTTP/SSE로 붙기만.

모바일 기존 Capacitor 그대로 확장

⚠️ 협업 주의 (중요)

"작업(Job) 저장 · 기기 레지스트리 · 푸시 · 오케스트레이터"는 내 담당(면접/파일) 밖의 공통 영역이다. AI 오케스트레이터는 공통 엔진, 스키마·인증도 공통.

→ 이 토대는 나 혼자 못 정한다. 팀 합의 먼저. 데스크탑 클라이언트 자체와 면접 도메인 연동은 내가 끌고 가도, 디스패치 토대는 팀 안건으로 올린다.
