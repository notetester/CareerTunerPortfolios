# 기획 문서 안내

이 폴더는 CareerTuner의 목표 상태, 제품 정책, UX 원칙과 단계별 로드맵을 설명한다. 기획 문서는
“왜 필요한가”와 “어디까지 만들 것인가”의 정본일 수 있지만, 현재 코드에 구현됐다는 증거는 아니다.
현재 상태는 런타임 소스와 모듈 `README.md`, [검증 원장](../verification/README.md)에서 확인한다.

## 작성 원칙

- 문서 상단에서 `현재`, `목표`, `결정 대기`, `폐기`를 구분한다.
- 일정이나 “최신” 상태를 본문 여러 곳에 덧붙이지 말고 상태 요약 한 곳을 갱신한다.
- 기술 스택·API·DB 경계는 [ARCHITECTURE.md](../ARCHITECTURE.md), 사용자 메뉴는
  [PRODUCT_STRUCTURE.md](../PRODUCT_STRUCTURE.md)를 우선한다.
- 목표가 구현되면 기획 문서를 운영 runbook으로 바꾸지 않는다. 구현 상태는 해당 모듈 문서에 연결하고,
  완료된 결정 과정은 [archive/](../archive/README.md)로 보낸다.
- 새 파일명은 소문자 ASCII `kebab-case`를 사용한다. 기존 한글·공백 경로는 링크 이전을 동반하는 별도
  작업에서만 변경한다.

## 최신성 확인

기획을 갱신할 때는 관련 런타임 문서와 실제 설정을 함께 확인한다.

1. 제품 범위: `기획.md`와 `PRODUCT_STRUCTURE.md`
2. 기술 가능 여부: `ARCHITECTURE.md`, `backend/README.md`, `frontend/README.md`
3. 플랫폼 상태: `frontend/MOBILE_BUILD.md`, `desktop/README.md`
4. AI 상태: `ml/*/README.md`, model card, AI 보고서·artifact 서브모듈
5. 검증 여부: `verification/`의 기준 SHA와 실행 증거
