## 1. 개요

CareerTuner는 채용공고에 맞춰 면접·스펙을 준비하는 AI 취업 플랫폼이다. 모바일 앱(Capacitor 기반 Android/iOS)은
같은 웹 코드베이스를 WebView로 감싼 형태이며, 음성 모의면접·아바타 화상 면접에서 **마이크·카메라**와
**음성/영상 데이터**를 처리한다. 본 문서는 그 처리 기준과 사용자 고지·동의 흐름을 정의한다.

## 2. 수집하는 개인정보

| 구분 | 항목 | 목적 | 담당 |
| --- | --- | --- | --- |
| 계정 | 이메일, 비밀번호(해시) | 로그인·식별 | A |
| 프로필 | 이력서·자기소개서·기술스택·경력 | AI 분석·면접 질문 생성 | A |
| **면접 답변** | **답변 텍스트, 음성 녹음, 영상 녹화** | **AI 채점·비언어 평가·리포트** | **D** |
| 결제 | 결제 식별자, 크레딧 사용 내역 | 구독·과금 | E |
| 커뮤니티 | 게시글·댓글 | 후기 공유·검열 | F |
| 사용량 로그 | AI 호출 기록(`ai_usage_log`) | 사용량·과금 정산 | 공통 |

> 원문(면접 답변·자기소개서·공고)은 보존하고, AI 분석 결과는 **별도 결과/이력으로** 저장한다(원본 미덮어쓰기).

## 3. 앱 권한 (Android / iOS)

| 권한 | 사용 기능 | 필요 이유 | 거부 시 동작 |
| --- | --- | --- | --- |
| **마이크** | 음성 모의면접, 실시간 음성 대화 | 음성 녹음 → STT 전사 + 전달력(톤·속도) 분석 | 음성·아바타 면접 불가. **텍스트 면접은 정상 사용** |
| **카메라** | 아바타 화상 면접 | 표정·자세 비언어 분석(MediaPipe) | 아바타 면접 불가. 음성/텍스트 면접은 가능 |
| 저장소(선택) | 결과 리포트·녹화 내보내기 | 사용자가 직접 다운로드할 때만 | 내보내기만 불가 |
| 알림(선택) | 학습 리마인더·결과 알림 | 푸시 발송 | 알림만 미수신 |

- 권한은 **해당 기능 진입 시점에 런타임 요청**한다(앱 최초 실행에 일괄 요청하지 않는다).
- Android: `AndroidManifest.xml`의 `RECORD_AUDIO`·`CAMERA` + Capacitor 권한 플러그인. iOS: `NSMicrophoneUsageDescription`·`NSCameraUsageDescription` 사용 목적 문구.

## 4. 영상·음성 데이터 처리 (★ 민감 — 핵심)

- **사전 동의**: 음성/영상 면접 시작 전 **동의 모달**로 "원본이 분석 서버로 전송됨"을 고지하고 동의를 받는다.
- **처리 방식**: 종료 후 원본을 분석 서버로 전송 → 서버에서 피처 추출 + 평가(ADR-006, 서버 Python 추론).
- **보관**: **점수(JSON)만 저장하고 음성·영상 원본은 분석 후 보관하지 않는다.** 사용자가 원하면 직접 다운로드.
- **동의 철회·미동의**: AI 데이터 활용에 동의하지 않거나 철회한 사용자는 해당 선택 기능을 실행하지 않는다.

## 5. 제3자 처리 위탁 (외부 AI 서비스)

| 수탁자 | 전달 데이터 | 용도 | 비고 |
| --- | --- | --- | --- |
| **자체 LLM(원격 GPU)** | 면접 텍스트 | 질문·채점 1차 처리 | **외부 인터넷 전송 아님**(자체 서빙) |
| OpenAI | 면접 텍스트 | 자체 LLM 실패 시 폴백 평가/생성 | 폴백 한정 |
| HeyGen (LiveAvatar) | 아바타 렌더용 음성/세션 | 화상 면접관 얼굴 렌더 | 자체 생성 비현실 → 외부 유지 |
| OpenAI Realtime | 실시간 음성 | 프리미엄 실시간 음성 대화 | 요금제 한정 |

> 비언어 평가(표정·음성)는 **자체 LightGBM 모델**로 서버 내 처리하며 외부 API로 보내지 않는다.

## 6. 정보주체 권리

- 열람·정정·삭제·처리정지 요청 가능.
- AI 평가 결과는 **참고 자료**이며 합격/불합격을 단정하지 않는다(코칭 보조 포지셔닝).
- 학습용 데이터에는 실제 개인정보를 비식별화 없이 넣지 않는다(합성 데이터는 파일로 분리, 운영 DB 미혼입).

## 7. 동의 흐름 (앱 UX)

1. **앱 최초 실행**: 필수(이용약관·개인정보처리방침) + 선택(AI 데이터 활용·마케팅) 동의 화면.
2. **기능별 동의**: 음성/영상 면접 진입 시 → 권한 요청 + 원본 전송 동의 모달(4장).
3. 동의 이력은 사용자별로 저장하고, 설정에서 언제든 철회할 수 있다.
