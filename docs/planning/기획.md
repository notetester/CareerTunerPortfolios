# 기획 — AI 취업 전략·면접 준비 플랫폼

좋아. 이 방향이면 사이트의 정체성은 **구인구직 플랫폼**이 아니라 **AI 취업 전략·면접 준비 플랫폼**으로 잡는 게 맞아.

즉, 사람인·잡코리아처럼 “공고를 모아 보여주는 사이트”가 아니라,

> 사용자가 가져온 채용공고와 본인 스펙을 AI가 분석해서
> **지원 전략, 부족 역량, 학습 방향, 예상 질문, 가상 면접, 답변 첨삭, 장기 취업 경향 분석**을 제공하는 사이트

로 가는 게 좋다.

---

# 1. 사이트 핵심 컨셉

## 프로젝트 주제명 후보

가장 적절한 제목은 이쪽이야.

> **AI 기반 채용공고 분석 및 맞춤형 면접 지원 플랫폼**

조금 더 기술적으로 쓰면,

> **채용공고·지원자 프로필 분석을 통한 AI 기반 취업 전략 및 가상 면접 평가 플랫폼**

React를 제목에 넣고 싶으면,

> **React 기반 AI 채용공고 분석 및 가상 면접 지원 서비스**

# 2. 서비스의 핵심 구조

이 사이트의 핵심 단위는 **공고**가 아니라 **지원 건**으로 잡는 게 좋다.

## 핵심 데이터 단위: 지원 건

예를 들면 다음과 같다.

```text
A유저
 ├─ B기업 / 2026-07-01 공고 / 사이트 개발직
 ├─ B기업 / 2026-07-01 공고 / 서버 관리직
 ├─ B기업 / 2026-07-20 공고 / 사이트 개발직
 ├─ C기업 / 2026-07-01 공고 / 전산직
 └─ D기업 / 수시채용 / 프론트엔드 개발직
```

이 각각이 ChatGPT의 채팅 세션처럼 따로 관리되는 구조가 된다.

각 지원 건 안에는 다음 정보가 들어간다.

| 항목      | 내용                         |
| ------- | -------------------------- |
| 기업 정보   | 기업명, 산업군, 최근 이슈, 주요 서비스    |
| 공고 정보   | 직무명, 채용일자, 요구 조건, 우대 조건    |
| 사용자 정보  | 이력서, 자기소개서, 경력, 기술스택, 자격증  |
| AI 분석   | 직무 적합도, 부족 역량, 준비 전략       |
| 면접 질문   | 예상 질문, 기술 질문, 인성 질문, 상황 질문 |
| 모의면접 기록 | 질문, 사용자 답변, 음성 기록, 평가 결과   |
| 첨삭 결과   | 개선 답변, 보완 포인트, 다음 연습 과제    |

즉, 사이트 구조는 **“공고별 AI 프로젝트 공간”**에 가깝다.

# 4. 사용 흐름

사이트의 기본 사용 흐름은 이렇게 잡으면 된다.

```text
1. 회원가입 / 로그인
2. 내 프로필 작성
3. 이력서, 자기소개서, 포트폴리오 업로드
4. 새 지원 건 생성
5. 채용공고 업로드 또는 붙여넣기
6. AI가 공고문 분석
7. AI가 기업 정보 조사
8. 내 스펙과 공고 요구조건 비교
9. 부족한 역량과 준비 전략 제공
10. 예상 면접 질문 생성
11. AI 가상 면접 진행
12. 답변 내용, 음성, 표정, 자세 분석
13. 면접 평가 리포트 제공
14. 개선 답변 및 학습 과제 추천
15. 여러 지원 건을 종합해 취업 경향 분석
```

여기서 가장 중요한 부분은 **지원 건 단위 분석**과 **사용자 전체 프로필 분석**을 분리하는 것이다.

# 5. 지원 건 상세 화면 구조

지원 건 하나를 열었을 때 화면은 ChatGPT 세션처럼 구성하면 좋다.

## 예시: B기업 / 사이트 개발직 / 2026-07-01 공고

```text
[좌측 사이드바]
- 전체 지원 건 목록
- 최근 분석한 공고
- 즐겨찾기
- 보관함

[중앙 메인 화면]
B기업 사이트 개발직 지원 준비

탭 1. 공고 분석
탭 2. 기업 분석
탭 3. 내 스펙 비교
탭 4. 지원 전략
탭 5. 예상 질문
탭 6. 가상 면접
탭 7. 면접 리포트
탭 8. 첨삭 기록

[우측 패널]
- 현재 준비도 점수
- 부족 역량
- 추천 학습
- 다음 할 일
- 결제/크레딧 사용량
```

이 구조가 좋은 이유는, 지원자가 한 공고를 준비할 때 필요한 정보가 전부 한 공간에 모이기 때문이다.

이 사이트에서 가장 중요한 기반 데이터다.

### 입력 항목

| 분류    | 내용                                 |
| ----- | ---------------------------------- |
| 기본 정보 | 이름, 희망 직무, 희망 산업군                  |
| 학력    | 학교, 전공, 졸업 여부                      |
| 경력    | 회사, 직무, 기간, 담당 업무                  |
| 프로젝트  | 프로젝트명, 역할, 기술스택, 성과                |
| 기술스택  | Java, React, Spring, Python, AWS 등 |
| 자격증   | 정보처리기사, SQLD, AWS 자격증 등            |
| 어학    | TOEIC, OPIC, TOEFL 등               |
| 자기소개서 | 문항별 자기소개서                          |

## 6.3 공고문 업로드 기능

이 사이트는 직접 공고를 수집하지 않고, 사용자가 직접 가져온 공고를 분석하는 방식이 적절하다.

### 입력 방식

| 방식       | 설명                    |
| -------- | --------------------- |
| 텍스트 붙여넣기 | 공고 내용을 복사해서 붙여넣기      |
| PDF 업로드  | 기업 채용 PDF 분석          |
| 이미지 업로드  | 캡처 이미지 OCR 분석         |
| URL 입력   | 사용자가 제공한 URL을 기반으로 분석 |
| 수동 입력    | 기업명, 직무명, 요구조건 직접 입력  |

다만 URL 입력은 조심해야 한다. 사이트가 외부 채용 플랫폼의 내용을 자동 수집하는 구조처럼 보이면 문제가 될 수 있으므로, 기본은 **사용자 업로드·사용자 입력 중심**으로 가는 게 안전하다.

## 6.5 기업 현황 조사

사용자가 공고문을 올리면 AI가 기업에 대해 추가 조사할 수 있다.

다만 이 기능은 선택형으로 두는 게 좋다.

```text
[기업 현황 조사하기]
- 기업 주요 사업
- 최근 뉴스
- 서비스/제품
- 경쟁사
- 채용 방향
- 면접에서 언급하면 좋은 포인트
```

예를 들어 사용자가 B기업 사이트 개발직에 지원한다면 AI는 이런 식으로 정리할 수 있다.

```text
이 기업은 최근 웹 서비스 고도화와 사용자 경험 개선에 집중하고 있으므로,
면접에서는 단순한 React 사용 경험보다
성능 최적화, 반응형 UI, 유지보수 가능한 컴포넌트 설계 경험을 강조하는 것이 좋습니다.
```

이 기능은 강력하지만, AI가 최신 정보를 조사하려면 외부 검색 비용이 들어가므로 **유료 기능**으로 빼도 좋다.

## 6.6 내 스펙과 공고 비교

이 사이트의 핵심 기능 중 하나다.

AI가 공고 요구조건과 사용자의 프로필을 비교한다.

### 예시

| 공고 요구조건     | 사용자 보유 여부 | 평가    |
| ----------- | --------: | ----- |
| React       |        보유 | 강점    |
| TypeScript  |     일부 경험 | 보완 필요 |
| AWS         |        없음 | 학습 필요 |
| Git 협업      |        보유 | 강점    |
| REST API 연동 |        보유 | 강점    |
| 포트폴리오       |        부족 | 보완 필요 |

그리고 다음처럼 종합한다.

```text
현재 지원자의 직무 적합도는 약 72점입니다.

강점:
- React 프로젝트 경험 보유
- REST API 연동 경험 보유
- Git 기반 협업 경험 보유

부족한 점:
- TypeScript 경험이 약함
- AWS 배포 경험이 부족함
- 포트폴리오 설명이 구체적이지 않음

우선 준비할 것:
1. React 프로젝트에서 본인이 맡은 역할 정리
2. TypeScript 기본 문법과 적용 사례 학습
3. AWS 배포 경험을 간단한 토이 프로젝트로 보완
4. 포트폴리오에 문제 해결 사례 추가
```

# 7. AI 가상 면접 구조

이 서비스의 대표 기능은 결국 **가상 면접**이다.

## 면접 모드

| 모드          | 설명              |
| ----------- | --------------- |
| 기본 면접       | 자기소개, 지원동기, 장단점 |
| 직무 면접       | 공고 기반 기술/직무 질문  |
| 인성 면접       | 협업, 갈등, 책임감, 태도 |
| 압박 면접       | 꼬리 질문, 반박 질문    |
| 실전 면접       | 시간 제한, 랜덤 질문    |
| 자기소개서 기반 면접 | 자소서 문장을 기반으로 질문 |
| 포트폴리오 기반 면접 | 프로젝트 설명 중심 질문   |
| 기업 맞춤 면접    | 기업 현황과 공고 기반 질문 |

---

## 면접 진행 방식

```text
1. 사용자가 면접 모드 선택
2. AI가 면접관 아바타 출력
3. AI가 음성으로 질문
4. 사용자가 마이크로 답변
5. 음성 인식으로 텍스트 변환
6. AI가 답변 내용 분석
7. 필요하면 꼬리 질문 생성
8. 면접 종료 후 전체 리포트 제공
```

---

## 면접 평가 항목

| 평가 영역  | 평가 내용                |
| ------ | -------------------- |
| 답변 내용  | 질문에 제대로 답했는가         |
| 직무 적합성 | 공고 요구 역량과 연결되는가      |
| 구체성    | 경험, 수치, 사례가 있는가      |
| 논리성    | 답변 구조가 자연스러운가        |
| 표현력    | 말이 명확한가              |
| 자신감    | 답변이 지나치게 위축되어 있지 않은가 |
| 태도     | 면접 상황에 적절한 태도인가      |
| 시간 관리  | 답변 시간이 너무 짧거나 길지 않은가 |
| 개선 가능성 | 어떤 식으로 고치면 좋은가       |

---

## 답변 첨삭 예시

```text
질문:
React 프로젝트 경험을 설명해주세요.

사용자 답변:
학교 프로젝트에서 React를 사용해서 게시판을 만들었습니다.

AI 평가:
답변의 방향은 맞지만 구체성이 부족합니다.
어떤 기능을 구현했는지, 본인이 맡은 역할이 무엇인지,
어떤 문제를 해결했는지가 드러나지 않습니다.

개선 방향:
- 프로젝트 목적 설명
- 본인 역할 설명
- 사용한 기술 설명
- 문제 해결 경험 추가
- 결과 또는 배운 점 제시

개선 답변:
학교 팀 프로젝트에서 React를 활용해 게시판 기능을 구현했습니다.
저는 게시글 목록 조회, 작성, 수정, 삭제 화면을 담당했고,
REST API를 통해 백엔드 서버와 데이터를 연동했습니다.
특히 useState와 useEffect를 사용해 게시글 목록을 갱신했고,
입력값 검증을 추가해 잘못된 데이터가 전송되지 않도록 처리했습니다.
이 경험을 통해 React 컴포넌트 구조와 API 연동 흐름을 이해할 수 있었습니다.
```

# 8. 음성·얼굴·몸 분석 기능

이 부분은 멀티모달 AI 기능으로 넣을 수 있다.

다만 표현은 조심하는 게 좋다.
“얼굴 인식”보다는 **표정·시선·자세 분석**이라고 하는 게 더 안전하고 자연스럽다.

## 분석 항목

| 분석 대상  | 기능                        |
| ------ | ------------------------- |
| 음성     | 말속도, 침묵 시간, 발화 길이, 목소리 크기 |
| 답변 텍스트 | 논리성, 직무 적합성, 구체성          |
| 표정     | 긴장, 무표정, 과도한 표정 변화        |
| 시선     | 카메라 응시 여부                 |
| 자세     | 몸 기울어짐, 흔들림               |
| 행동     | 손동작 과다, 고개 숙임 등           |

주의할 점은, 표정이나 자세 분석을 **합격/불합격 판단 근거**처럼 쓰면 안 된다는 것이다.
이 기능은 어디까지나 “면접 태도 개선 참고자료”로 쓰는 게 좋다.

예를 들면,

```text
시선이 자주 아래로 향했습니다.
실제 면접에서는 답변 중간중간 면접관을 바라보는 연습이 필요합니다.
```

정도는 괜찮지만,

```text
표정이 어두우므로 직무 적합성이 낮습니다.
```

이런 식의 평가는 위험하다.

# 9. 전체 취업 경향 분석

네가 말한 “다른 채팅 내역을 참조해서 이후 채팅에 영향을 주는 구조”는 이 사이트에서 매우 좋은 기능이 될 수 있다.

다만 구현상으로는 다음 두 계층으로 나누면 된다.

## 1단계: 지원 건별 분석

각 공고마다 독립 분석한다.

```text
B기업 사이트 개발직 → React, TypeScript, UI 개발 중심
C기업 전산직 → 네트워크, 서버, 정보처리기사 중심
D기업 백엔드직 → Java, Spring, DB 중심
```

## 2단계: 사용자 전체 취업 경향 분석

여러 지원 건을 종합한다.

```text
최근 10개 지원 건 분석 결과:
- 프론트엔드/웹 개발 직무 지원 비율이 높음
- React 관련 공고를 자주 선택함
- TypeScript와 AWS가 반복적으로 부족 역량으로 나타남
- 면접 답변에서 프로젝트 성과 설명이 약함
- 기술 경험은 있으나 수치화된 결과 제시가 부족함
```

이걸 바탕으로 AI가 장기 전략을 추천한다.

```text
추천 전략:
1. 프론트엔드 직무 중심으로 지원 방향을 좁히는 것이 좋습니다.
2. TypeScript 학습을 우선순위로 두는 것이 좋습니다.
3. AWS 배포 경험을 포트폴리오에 추가하면 다수 공고에서 경쟁력이 올라갑니다.
4. 면접에서는 단순히 “만들었다”보다 “어떤 문제를 해결했다”를 중심으로 답변을 구성해야 합니다.
```

이 기능은 서비스 차별화에 상당히 좋다.

# 19. 최종 사이트 구조 요약


# 20. 한 문장 소개

프로젝트 소개문은 이렇게 쓰면 좋다.

> 본 서비스는 사용자가 직접 업로드한 채용공고와 개인 이력 정보를 기반으로 AI가 직무 요구사항, 우대 조건, 기업 현황을 분석하고, 지원자의 역량과 비교하여 맞춤형 지원 전략, 필요 학습 방향, 자격증 추천, 예상 면접 질문을 제공하는 취업 지원 플랫폼이다. 또한 AI 가상 면접관을 통해 모의면접을 진행하고, 사용자의 답변 내용과 음성·태도 정보를 분석하여 면접 평가, 답변 첨삭, 개선 방향을 제시한다.

현재 단계에서는 **“지원 건 관리 + 공고 분석 + 스펙 비교 + AI 모의면접 + 답변 첨삭”**을 1차 핵심 기능으로 잡는 게 가장 현실적이다.
음성, 아바타, 표정·자세 분석은 2차·3차 확장 기능으로 두는 게 좋다.
