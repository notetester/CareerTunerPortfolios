첨부한 ZIP 기준으로 보면, 디자인 초안은 **생각보다 꽤 잘 잡혀 있다.**
단순 랜딩페이지가 아니라 `Home / Dashboard / Applications / ApplicationDetail / AIInterview / Analysis / Community / Pricing / Profile / Login`까지 들어가 있어서, 이미 **서비스 전체 구조를 보여주는 프로토타입**에 가깝다.

다만 이름은 바꾸는 게 맞다. `JobPrep`은 너무 일반적이고, 이미 유사 명칭·유사 서비스가 많다. 실제로 검색해보면 `Fiterview`는 포트폴리오와 채용공고를 함께 분석해 맞춤 면접 질문을 생성하고 피드백을 제공하는 서비스로 소개되고 있고, `잡터뷰`도 맞춤형 AI 모의면접 앱으로 확인된다. 사람인도 이미 AI 모의면접, AI 서류합격 코칭 같은 기능을 제공하고 있다. 따라서 이름과 포지셔닝에서 **“그냥 AI 면접 앱”이 아니라 “공고 업로드 기반 지원 전략 플랫폼”**이라는 점을 더 강하게 잡아야 한다. ([Fiterview][1])

# 1. 전체 평가

## 결론부터 말하면

**디자인 방향은 좋다.**
특히 네가 말했던 핵심 구조인,

> 지원 건별 관리 → 공고 분석 → 스펙 비교 → 전략 추천 → 예상 질문 → AI 면접 → 첨삭 → 장기 취업 경향 분석

이 흐름이 화면에 잘 반영되어 있다.

현재 초안의 가장 좋은 점은 **“지원 건”을 서비스의 중심 단위로 잡았다는 것**이다. 이게 사람인·잡코리아·인크루트와 다른 정체성을 만든다.

기존 취업 플랫폼은 공고를 보여주는 곳이고, 이 서비스는 사용자가 가져온 공고를 바탕으로 **지원 준비를 설계하는 곳**이다. 이 차이가 잘 살아 있다.

# 2. 가장 잘 만든 부분

## 1) `ApplicationDetail` 화면이 핵심을 잘 잡았다

지원 건 상세 화면이 가장 좋다.

좌측에 지원 건 목록이 있고, 중앙에는 공고 분석·기업 분석·스펙 비교·지원 전략·예상 질문·가상 면접·면접 리포트·첨삭 기록 탭이 있고, 우측에는 준비도 점수·부족 역량·추천 학습·다음 할 일·크레딧 사용량이 있다.

이 구조는 거의 그대로 가져가도 된다.

특히 좋은 점은 이거다.

```text
좌측: 여러 지원 건 목록
중앙: 현재 지원 건의 상세 분석
우측: 현재 준비도와 다음 행동
```

이건 ChatGPT의 세션 구조와 취업 준비 서비스를 잘 결합한 형태다.

---

## 2) `Dashboard`도 서비스 성격을 잘 보여준다

대시보드에 다음 요소들이 들어가 있다.

* 활성 지원 건
* 총 모의면접 횟수
* 보유 크레딧
* 평균 적합도
* 최근 지원 건
* 오늘의 할 일
* 최근 활동
* 부족 역량

이건 사용자가 로그인 후 가장 먼저 봐야 할 정보로 적절하다.

다만 현재 예시 데이터가 `카카오페이`, `네이버`, `삼성SDS`, `React`, `Spring`, `TypeScript` 중심이라서 서비스가 **개발자 취업 전용 사이트처럼 보일 위험**이 있다.

서비스직에서 벗어나 범용 취업 지원 플랫폼으로 가려는 것이니, 예시 데이터는 여러 직군으로 섞는 게 좋다.

예를 들면:

```text
카카오페이 · 프론트엔드 개발자
CJ올리브영 · 마케팅 기획
현대자동차 · 생산관리
국민건강보험공단 · 전산직
스타트업 A · 서비스 운영 매니저
```

이렇게 해야 “개발자용 AI 면접 사이트”가 아니라 “전체 취업 준비 플랫폼”처럼 보인다.

---

## 3) `Pricing` 화면이 현실적이다

요금제를 월 구독형과 크레딧형으로 나눈 것은 좋다.

AI 서비스는 기능마다 비용 차이가 크다.

* 공고 분석
* 기업 현황 조사
* 예상 질문 생성
* 텍스트 모의면접
* 음성 모의면접
* 영상/자세 분석
* 자기소개서 첨삭

이 기능들을 전부 월정액 무제한으로 주면 비용 통제가 어렵다. 그래서 크레딧형을 같이 둔 것은 좋은 판단이다.

다만 “무제한”이라는 표현은 조심해야 한다. AI 비용이 실제로 발생하므로 나중에는 다음처럼 바꾸는 게 안전하다.

```text
무제한 → 월 100회
무제한 → 합리적 사용량 내 제공
무제한 → 일일 사용량 제한 있음
```

# 3. 보완해야 할 부분

## 1) 랜딩페이지가 너무 많은 내용을 한 번에 말한다

`Home` 화면은 기능 설명이 풍부해서 좋지만, 현재는 너무 많은 것을 한 페이지에서 보여준다.

현재 들어간 요소는 대략 이렇다.

* 히어로
* 8가지 취업 솔루션
* 12단계 프로세스
* 지원 건 공간 설명
* 면접 모드 설명
* 일반 채용 사이트와 비교
* 요금제
* 후기
* CTA

서비스 초안으로는 좋지만, 실제 랜딩페이지에서는 정보량이 조금 많다.

첫 화면에서 가장 먼저 전달해야 할 메시지는 하나면 충분하다.

> 채용공고를 올리면 AI가 내 스펙과 비교해서 지원 전략과 면접 준비를 만들어준다.

이 문장이 3초 안에 보여야 한다.

추천 랜딩 구조는 이렇게 줄이는 게 좋다.

```text
1. 히어로: 공고 업로드 → AI 지원 전략
2. 핵심 흐름 4단계
   공고 분석 / 스펙 비교 / 예상 질문 / 모의면접
3. 지원 건 상세 예시
4. AI 면접 리포트 예시
5. 요금제
6. 시작하기
```

“12단계 프로세스”는 서비스 소개 페이지나 기능 상세 페이지로 빼도 된다.

---

## 2) 이름 때문에 서비스가 다소 평범해 보인다

`AI JobPrep`은 의미는 직관적인데, 너무 일반적이다.

문제는 두 가지다.

첫째, `JobPrep` 자체가 너무 흔한 단어 조합이다.
둘째, 이 서비스의 진짜 차별점인 **공고 기반 지원 전략**이 이름에 드러나지 않는다.

이 서비스는 단순히 면접 연습만 하는 서비스가 아니다.

핵심은 이것이다.

```text
공고를 가져온다
→ 내 스펙과 비교한다
→ 부족한 점을 찾는다
→ 전략을 만든다
→ 예상 질문을 만든다
→ 면접을 연습한다
→ 답변을 고친다
```

따라서 이름도 `JobPrep`보다 **공고, 지원, 핏, 전략, 준비도, 면접** 중 하나를 잡는 게 낫다.

---

## 3) 상단 메뉴가 조금 과하다

현재 헤더 메뉴가 다음처럼 많다.

```text
대시보드
내 프로필
지원 건 관리
AI 가상 면접
AI 첨삭
취업 분석
커뮤니티
```

기능적으로는 맞지만, 처음 보는 사용자는 조금 복잡하게 느낄 수 있다.

추천 구조는 이쪽이다.

```text
대시보드
지원 건
AI 면접
분석 리포트
커뮤니티
요금제
```

`내 프로필`, `AI 첨삭`, `설정`은 상단 메뉴보다 사용자 메뉴나 사이드 메뉴에 넣는 게 낫다.

특히 `AI 첨삭`은 독립 메뉴로 두기보다 `지원 건 상세` 안에 들어가는 게 자연스럽다. 사용자가 첨삭을 따로 하러 오는 게 아니라, 특정 지원 건을 준비하다가 답변이나 자소서를 첨삭받는 흐름이기 때문이다.

---

## 4) 실제 구현상 쿼리 탭 처리가 아직 안 잡혀 있다

코드 구조를 보면 `/profile?tab=resume`, `/interview?tab=report`, `/applications?tab=fit` 같은 URL이 메뉴에 들어가 있지만, 실제로 `useSearchParams`나 `location.search`를 읽는 처리는 아직 보이지 않는다.

즉, 지금 상태에서는 사용자가 메뉴에서 “이력서 관리”를 눌러도 실제 탭이 자동으로 열리지 않을 가능성이 높다.

나중에 구현할 때는 이런 식으로 처리해야 한다.

```text
/profile?tab=resume → 이력서 관리 탭 활성화
/interview?tab=report → 면접 리포트 탭 활성화
/pricing?tab=credits → 크레딧형 탭 활성화
```

---

## 5) `/applications/new` 처리가 애매하다

현재 라우트는 `/applications/:id`가 있으므로 `/applications/new`도 상세 페이지로 들어갈 수 있다.

프로토타입 단계에서는 괜찮지만, 실제 서비스에서는 다음 중 하나로 정리하는 게 좋다.

```text
/applications/new → 새 지원 건 생성 페이지
/applications/:id → 기존 지원 건 상세 페이지
```

또는 모달 방식이면:

```text
/applications?action=new
```

둘 중 하나만 쓰는 게 좋다. 지금은 둘이 섞여 있다.

---

## 6) 영상·표정·자세 분석은 표현을 조심해야 한다

디자인에 영상/자세 분석 면접 기능이 들어가는 것은 좋다. 다만 서비스 문구에서는 “얼굴 인식”보다는 다음 표현이 낫다.

```text
표정·시선·자세 분석
```

“얼굴 인식”은 개인 식별, 생체정보, 개인정보 이슈가 커진다. 반면 “면접 태도 개선을 위한 시선·자세 분석”이라고 하면 기능 목적이 명확해진다.

또한 분석 결과도 조심해야 한다.

좋은 표현:

```text
답변 중 시선이 자주 아래로 향했습니다.
실제 면접에서는 카메라 또는 면접관을 바라보는 연습이 필요합니다.
```

위험한 표현:

```text
표정이 어두우므로 직무 적합성이 낮습니다.
```

자세·표정 분석은 합격 가능성 판단이 아니라 **면접 태도 개선 참고자료**로 제한하는 게 좋다.

# 5. 사이트 이름 후보

이름은 크게 세 방향으로 잡을 수 있다.

## 방향 A: “공고 기반 분석”을 강조

이 서비스의 가장 큰 차별점은 사용자가 공고를 가져와서 분석한다는 점이다.

| 후보        | 느낌                      | 평가             |
| --------- | ----------------------- | -------------- |
| **공고핏터**  | 공고에 맞게 스펙과 답변을 맞춰주는 서비스 | 가장 추천          |
| **공고핏**   | 공고와 내 스펙의 적합도 분석        | 직관적이지만 조금 일반적  |
| **공고해석소** | 채용공고를 쉽게 풀어주는 곳         | 친근하고 설명적       |
| **공고분석관** | 공고를 분석해주는 AI 도구         | 공공기관 느낌이 조금 있음 |
| **공고매치**  | 공고와 나의 매칭               | 직관적이지만 흔할 수 있음 |

이 방향에서는 **공고핏터**가 가장 좋아 보인다.

```text
공고핏터
: 채용공고에 맞춰 내 스펙, 답변, 면접 전략을 조정해주는 AI 취업 지원 플랫폼
```

---

## 방향 B: “지원 전략”을 강조

이 사이트가 단순 면접 앱이 아니라 지원 전략 전체를 다룬다는 점을 강조하는 방향이다.

| 후보         | 느낌               | 평가                    |
| ---------- | ---------------- | --------------------- |
| **지원핏**    | 지원하려는 공고와 나의 적합도 | 짧고 좋음                 |
| **지원나침반**  | 어디에 어떻게 준비할지 안내  | 직관적이고 신뢰감 있음          |
| **지원전략소**  | 지원 전략을 설계하는 곳    | 설명적이고 안정적             |
| **합격설계소**  | 합격을 설계한다는 느낌     | 좋지만 “합격” 표현은 과장 느낌 가능 |
| **커리어설계소** | 장기 커리어 방향까지 포함   | 범용적이지만 조금 넓음          |

이 방향에서는 **지원핏**이나 **지원나침반**이 좋다.

다만 `합격`이 들어간 이름은 마케팅적으로 강하지만, 유료 AI 서비스에서는 “합격 보장”처럼 보일 수 있어서 조심하는 게 좋다.

---

## 방향 C: “면접 준비”를 강조

면접 기능을 전면에 내세우는 방향이다.

| 후보        | 느낌               | 평가              |
| --------- | ---------------- | --------------- |
| **면접핏**   | 면접 답변을 공고에 맞게 조정 | 직관적             |
| **면접정비소** | 답변을 고치고 다듬는 공간   | 참신하지만 약간 캐주얼    |
| **인터뷰핏**  | 면접 적합도와 답변 개선    | 좋지만 유사명 많을 수 있음 |
| **답변핏**   | 면접 답변 첨삭 중심      | 기능이 좁아 보임       |
| **면접루트**  | 면접 준비 경로 제공      | 무난함             |

이 방향은 좋지만, 서비스 전체를 담기에는 조금 좁다.
이 서비스는 면접만 하는 게 아니라 공고 분석과 스펙 비교가 핵심이므로 이름을 면접에만 묶는 것은 아깝다.

---

## 방향 D: 영어/브랜드형 이름

| 후보              | 의미               | 평가             |
| --------------- | ---------------- | -------------- |
| **ApplyMap**    | 지원 경로를 지도처럼 안내   | 직관적            |
| **SpecPilot**   | 스펙 준비를 조종해주는 AI  | 괜찮음            |
| **CareerTuner** | 커리어·스펙·답변을 조정    | 세련됨            |
| **OfferPath**   | 합격/오퍼까지 가는 경로    | 괜찮지만 해외 서비스 느낌 |
| **FitPilot**    | 적합도 기반 취업 준비 도우미 | 짧고 무난          |
| **Prepilot**    | Prep + Pilot     | 참신하지만 뜻 설명이 필요 |
| **ApplyScope**  | 지원 공고를 분석해보는 렌즈  | 괜찮음            |
| **RoleMap**     | 직무별 준비 경로        | 직무 분석 느낌 강함    |

영어형은 세련되지만, 국내 취준생 대상이면 한글 이름이 더 빠르게 이해된다.

# 6. 내가 고른 최종 추천 TOP 7

## 1위: 공고핏터

가장 추천한다.

이유는 명확하다.

* 공고 업로드 기반 서비스라는 차별점이 드러남
* “핏터”가 스펙과 답변을 맞춰주는 느낌을 줌
* 면접뿐 아니라 공고 분석, 스펙 비교, 전략 추천까지 포함 가능

브랜드 문구:

```text
공고핏터
채용공고에 맞춰 내 스펙과 면접 답변을 조정하는 AI 취업 전략 플랫폼
```

단점은 공고 분석, 스펙 비교, 장기 취업 경향 분석까지 담기에는 이름이 좁다.
