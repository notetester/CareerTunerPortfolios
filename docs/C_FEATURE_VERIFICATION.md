# C 영역 기능 검증 원장 (Verification Ledger)

C 영역(취업 분석·적합도·자격증 근거·자체 모델)과 인접 회색지대의 **실제 동작 확인 이력**을 한곳에 기록한다.
목적: 매 작업마다 같은 확인을 반복하지 않고, ① 검증이 안 된 것 ② 검증했지만 이후 변경이 쌓여
재검증이 필요한 것을 즉시 식별한다.

**갱신 규칙**: 해당 기능을 바꾸는 PR을 머지하면 그 행의 '최종 검증'을 갱신하거나 상태를 `재검증 필요`로
내린다. 라이브 확인은 배포된 dev 환경(EC2) 기준. 상태 의미 — ✅ 검증됨(해당 시점 기준 현행) /
🔶 부분(일부 경로만 검증) / ⏳ 재검증 필요(검증 후 변경 누적) / ❌ 미구현(요구는 있으나 기능 없음).
