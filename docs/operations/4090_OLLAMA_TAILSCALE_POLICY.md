# 공용 GPU(4090) Ollama 운영 정책

> 4090 PC의 Ollama를 **팀 공용 모델 서버**로 운영하기 위한 공통 정책. 특정 파트 전용이 아니다.
> 이 문서는 **정책**만 다룬다. 특정 PC에 대한 일회성 실행 명령·접근 권한 부여 방식·노출 지속 운영 절차는 저장소에 두지 않고 별도(채팅/운영 런북)로 관리한다.

## 1. 팀 공용 모델 서버
- 4090 Ollama는 특정 파트 전용이 아니라 **팀 공용**이다. C/D 및 필요 시 다른 파트 모델이 같은 Ollama 서버(11434)에 공존할 수 있다.
- 하나의 Ollama에 여러 파트 모델이 올라가므로, **바인딩·포트·모델 상태 변경은 다른 파트에 영향**을 준다.

## 2. 표준 접속 주소 / API 경로
- 팀 공통 표준 주소: `OSS_BASE_URL=http://localhost:11434/v1`
- 4090 **내부 로컬 디버깅** 예외: `OSS_BASE_URL=http://localhost:11434/v1`
- ⚠️ `/v1`은 모델 버전이 아니라 **OpenAI 호환 API 경로**다. 모델 선택은 URL이 아니라 `model` 값(파트별 환경변수)로 한다.
  - 잘못된 예: `http://localhost:11434/c3`, `.../a2`
  - 올바른 구조:
    ```text
    OSS_BASE_URL=http://localhost:11434/v1
    C_MODEL=careertuner-c-career-strategy-3b
    D_MODEL=interview-3b
    ```
- 즉 **주소는 공통, 모델명은 파트별로 분리**한다.

## 3. 바인딩 / 접근 제어 (개념 구분)
- 바인딩 정책: **localhost + VPN 접근을 모두 허용**하고, 일반 LAN 접근은 방화벽으로 **예방 차단**한다.
- 다음 둘은 서로 다른 계층이다. 혼동하지 않는다.

  | 구분 | 의미 |
  | --- | --- |
  | VPN(메시 VPN) 접근 권한 | `100.x` 경로에 **누가** 접근 가능한지 통제(저장소 외부 운영 관리) |
  | 서버 바인딩(`OLLAMA_HOST`) | Ollama가 **어느 네트워크 인터페이스**에서 요청을 받는지 |

- `localhost`으로 접근하려면 VPN 접근 권한이 필요하다. 반면 서버 바인딩이 넓으면 LAN 인터페이스까지 열릴 수 있으므로 **방화벽 제한을 함께** 둔다.
- 노출 범위·지속 운영의 **구체 적용 절차**는 저장소에 두지 않는다(운영 런북에서 관리).

## 4. 모델 변경 / 소유 관리
- 모델명은 개선·교체 과정에서 바뀔 수 있다. 단 **백엔드가 참조하는 대표 모델명**을 바꾸면, 해당 파트 담당이 **환경변수와 E2E 검증 결과를 함께 갱신**한다.
- 각 파트는 자기 파트 모델을 추가/교체할 수 있다. **타 파트 모델 임의 삭제/교체 금지.**
- 공용 Ollama 자체의 재시작·포트 변경·바인딩 변경은 팀 영향이 있으므로 **사전 공지 후 신중히** 처리한다.

## 5. 발표 / 리허설 동결 규칙
- 특정 파트에 리허설 **우선권은 없다.**
- 단, 발표/리허설 **직전**에는 공용 Ollama의 다음을 피한다(전체 데모 안정성):
  - 서버 재시작 · 포트 변경 · `OLLAMA_HOST` 변경 · 방화벽 변경 · 대표 모델명 변경 · 모델 삭제
- 이는 특정 파트 우선권이 아니라 **최소 동결 규칙**이다.

## 6. 동시 요청
- 동시 요청은 **허용**한다. 단 단일 공용 GPU 자원이라 동시 호출 시 응답 지연·모델 로딩 대기·VRAM 증가·모델 전환 지연이 생길 수 있다.
- 발표/리허설 중에는 **불필요한 반복 호출·부하 테스트성 요청을 자제**한다.
