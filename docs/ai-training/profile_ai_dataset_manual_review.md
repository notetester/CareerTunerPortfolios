# 프로필 AI 학습 데이터 수동 검수 보고서

검수일: 2026-06-23  
대상 파일: `docs/ai-training/profile_ai_seed_samples.jsonl`  
샘플 수: 30개

## 1. 결론

현재 30개 샘플은 **1차 학습 흐름 검증용 데이터셋으로 사용 가능**합니다.

다만 이 데이터셋은 아직 최종 품질의 대규모 학습 데이터가 아니라, 아래 목적에 적합한 시작 데이터입니다.

- 학습 데이터 형식 검증
- 직무군별 평가 기준 학습 흐름 확인
- 모델이 CareerTuner 응답 JSON 형식을 따르는지 확인
- 백엔드 `ProfileAiJsonValidator` 통과 가능성 확인
- 이후 100개 이상 데이터 확장을 위한 기준 샘플

## 2. 자동 검증 결과

명령:

```bash
python docs/ai-training/validate_profile_ai_dataset.py docs/ai-training/profile_ai_seed_samples.jsonl
```

결과:

```text
OK: 30 samples passed structural validation
Quality warnings: none
```

확인된 항목:

- JSONL 파싱 가능
- assistant 응답 JSON 파싱 가능
- 필수 필드 존재
- 6개 평가 기준 모두 존재
- `rawScore`가 0~100 범위
- 직무군 분포 기준 충족
- 개인정보 의심 패턴 없음
- 지나치게 추상적인 추천 문장 없음

## 4. 점수 분포 검수

| 평가 기준 | 평균 | 최솟값 | 최댓값 | 검수 판단 |
| --- | ---: | ---: | ---: | --- |
| `GOAL_CLARITY` | 80.9 | 42 | 86 | 적정 |
| `EXPERIENCE_SPECIFICITY` | 79.0 | 58 | 82 | 적정 |
| `ACHIEVEMENT_EVIDENCE` | 65.1 | 40 | 82 | 적정 |
| `JOB_SKILL_ALIGNMENT` | 81.0 | 48 | 88 | 적정 |
| `DOCUMENT_CONSISTENCY` | 75.8 | 60 | 84 | 적정 |
| `IMPROVEMENT_READINESS` | 82.0 | 76 | 86 | 적정 |

검수 의견:

- `ACHIEVEMENT_EVIDENCE` 평균이 가장 낮게 잡혀 있습니다. 이는 대부분의 신입/준비생 프로필이 경험은 있어도 성과 수치가 부족하다는 현실과 맞습니다.
- `IMPROVEMENT_READINESS` 평균이 높은 편입니다. 샘플들이 보완 방향을 제시하기 쉬운 형태로 작성되어 있기 때문에 자연스러운 분포입니다.
- `GENERAL` 샘플은 목표 명확성, 성과 근거, 직무 역량 적합도가 낮게 잡혀 있어 직무 미정 프로필에 대한 평가 기준으로 적절합니다.
- 모든 점수가 90점 이상인 과도한 고득점 샘플이 없고, 모든 점수가 과도하게 낮은 샘플도 없습니다.

## 5. 직무군별 내용 검수

### DEVELOPMENT_DATA

포함 샘플:

- 백엔드 개발자
- 데이터 분석가
- QA 엔지니어
- 프론트엔드 개발자
- 클라우드 인프라 엔지니어

검수 판단:

- 개발 직군 내부에서도 백엔드, 데이터, QA, 프론트엔드, 인프라로 나누어져 있어 편중이 적습니다.
- 기술 스택과 희망 직무가 잘 연결되어 있습니다.
- 성과 근거가 부족한 샘플은 `ACHIEVEMENT_EVIDENCE`를 낮게 부여해 점수 기준이 자연스럽습니다.

### SALES_MARKETING

포함 샘플:

- 콘텐츠 마케터
- B2B 영업관리
- CRM 마케터
- 퍼포먼스 마케터
- 브랜드 마케터

검수 판단:

- 영업, CRM, 퍼포먼스, 콘텐츠, 브랜드가 분리되어 있어 마케팅 내부 다양성이 있습니다.
- 도달률, 클릭률, 전환율, 리드, 설문 등 직무별 성과 지표를 추천 액션에 반영했습니다.
- 마케팅 샘플이 단순 콘텐츠 제작이 아니라 데이터 기반 개선 관점까지 포함하고 있어 CareerTuner 방향과 맞습니다.

### DESIGN_CONTENT

포함 샘플:

- UX/UI 디자이너
- 콘텐츠 디자이너
- 영상 편집자
- 브랜드 디자이너

검수 판단:

- 디자인 직무가 UX/UI에만 치우치지 않고 콘텐츠, 영상, 브랜드까지 포함됩니다.
- 추천 액션이 포트폴리오 개선 중심으로 구체적입니다.
- 개선 전후 지표, 사용자 반응, 디자인 의사결정 근거를 요구하도록 설계되어 실무형 피드백에 가깝습니다.

### BUSINESS_OFFICE

포함 샘플:

- 서비스 기획자
- HR 채용 담당자
- 회계 사무원
- 운영 매니저
- 상품 기획자

검수 판단:

- 사무/기획 계열 안에서도 기획, HR, 회계, 운영, 상품기획으로 분산되어 있습니다.
- 문서화, 체크리스트, 지표, 프로세스 개선 같은 사무직군 핵심 역량이 반영되어 있습니다.
- 추천 액션이 실제 산출물 작성으로 연결됩니다.

### HEALTHCARE_SERVICE

포함 샘플:

- 신입 간호사
- 병원 원무 행정
- 고객 상담원

검수 판단:

- 의료 직무와 서비스 직무가 함께 포함되어 있습니다.
- 환자 안전, 개인정보, 고객 응대, 접수 정확성 같은 도메인 특성이 반영되어 있습니다.
- 의료 전문 판단을 모델이 과도하게 수행하지 않고, 프로필 보완 중심으로 평가하고 있습니다.

### EDUCATION_PUBLIC

포함 샘플:

- 교육 운영 매니저
- 청소년 코딩 강사
- 공공기관 행정직
- 직업훈련 상담사

검수 판단:

- 교육, 강의, 공공행정, 상담 지원 영역이 고르게 들어가 있습니다.
- 수료율, 만족도, 공문, 상담 기록, 교육 과정 안내처럼 각 직무군의 산출물과 지표가 반영되어 있습니다.

### PRODUCTION_LOGISTICS

포함 샘플:

- 생산관리
- 물류 운영
- 품질관리

검수 판단:

- 생산, 물류, 품질이 분리되어 있습니다.
- 불량률, 처리량, 피킹 오류, HACCP 등 현장형 지표와 기준이 포함되어 있습니다.
- 사무직/개발직 관점이 아니라 현장 운영 관점의 피드백을 제공합니다.

### GENERAL

포함 샘플:

- 아직 희망 직무 미정

검수 판단:

- 직무가 불명확한 사용자를 위해 필요한 샘플입니다.
- 목표 명확성, 직무 역량 적합도, 성과 근거 점수가 낮게 잡혀 있어 자연스럽습니다.
- 추천 액션이 "관심 직무를 좁히기"부터 시작하므로 적절합니다.

## 6. 추천 액션 품질 검수

검수 기준:

- 사용자가 바로 수정할 수 있는가
- 너무 추상적인 표현을 피했는가
- 직무별 산출물과 지표를 반영했는가

검수 결과:

- 모든 샘플이 추천 액션 3개를 포함합니다.
- "더 노력하세요", "역량을 강화하세요" 같은 추상적 문장은 발견되지 않았습니다.
- 대부분의 추천이 표, 체크리스트, 포트폴리오, 지표, 사례 보강으로 연결됩니다.

대표적으로 좋은 패턴:

- 프로젝트 설명에 사용 기술, 담당 역할, 결과 수치를 추가하라고 안내
- 마케팅 샘플에서 클릭률, 전환율, 오픈율 같은 지표를 요구
- 디자인 샘플에서 Before/After, 사용자 반응, 디자인 의사결정 근거를 요구
- 운영/사무 샘플에서 체크리스트, 정책서, 프로세스 문서화를 요구
- 생산/물류 샘플에서 불량률, 처리량, 오류 유형을 요구

## 7. 개인정보 및 민감정보 검수

검수 결과:

- 실제 이메일 없음
- 전화번호 없음
- 주민등록번호 없음
- 실제 주소 없음
- 특정 개인 식별 정보 없음

주의:

- 샘플에는 가상의 학교명, 회사명, 포트폴리오 URL을 사용했습니다.
- 실제 학습 데이터로 확장할 때는 실제 사용자 이력서, 이메일, 전화번호, 회사 내부 정보가 들어가지 않도록 별도 마스킹이 필요합니다.

## 8. 현재 데이터셋의 한계

현재 30개 데이터셋은 1차 학습 흐름 검증용으로는 적합하지만, 실제 서비스 품질을 기대하기에는 아직 작습니다.

한계:

- 각 직무군당 데이터 수가 많지 않습니다.
- 같은 직무군 안에서도 경력 수준별 다양성이 아직 부족합니다.
- 실제 사용자 문체와 오탈자, 불완전 입력 케이스가 부족합니다.
- 매우 나쁜 프로필, 매우 우수한 프로필, 애매한 프로필의 극단값이 더 필요합니다.

## 9. 다음 확장 방향

1차 학습 성공 후에는 100개 이상으로 확장하는 것이 좋습니다.

추가해야 할 샘플 유형:

- 신입이지만 프로젝트가 거의 없는 프로필
- 경력은 있지만 직무 전환을 원하는 프로필
- 자격증은 많지만 경험이 부족한 프로필
- 성과 수치는 많지만 문서가 산만한 프로필
- 자기소개서만 있고 프로젝트가 없는 프로필
- 포트폴리오 링크는 있지만 설명이 부족한 프로필
- 희망 직무와 기술 스택이 맞지 않는 프로필
- 의료/상담처럼 민감한 도메인의 안전한 표현이 필요한 프로필

## 10. 최종 판단

현재 데이터셋은 아래 기준을 충족합니다.

- 구조 검증 통과
- 직무군 분포 적정
- 점수 분포 적정
- 추천 액션 구체성 적정
- 개인정보 위험 낮음
- CareerTuner A파트 백엔드 검증 구조와 호환 가능

따라서 다음 단계인 **학습 방식 선택 및 1차 학습 실행**으로 넘어갈 수 있습니다.
