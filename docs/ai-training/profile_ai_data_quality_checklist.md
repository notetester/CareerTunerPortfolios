# 프로필 AI 학습 데이터 품질 검수표

이 문서는 `profile_ai_seed_samples.jsonl` 또는 이후 확장될 학습 데이터가 정말 학습에 넣어도 되는 데이터인지 판단하기 위한 기준입니다.

## 1. 검증은 두 종류로 나눈다

학습 데이터 검증은 크게 두 가지입니다.

| 구분 | 의미 | 도구 |
| --- | --- | --- |
| 구조 검증 | JSONL 문법, 필수 필드, 평가 기준, 점수 범위가 맞는지 확인 | `validate_profile_ai_dataset.py` |
| 내용 검증 | 평가가 타당한지, 추천이 현실적인지, 직무군이 맞는지 사람이 확인 | 이 문서의 체크리스트 |

구조 검증을 통과했다고 해서 무조건 좋은 데이터는 아닙니다.
구조 검증은 "파일이 깨지지 않았다"는 뜻이고, 내용 검증은 "이 데이터가 학습할 만하다"는 뜻입니다.

## 2. 샘플 데이터 1개의 기준

샘플 데이터 1개는 아래 두 가지로 이루어집니다.

```text
사용자 프로필 입력 1개
+ CareerTuner가 원하는 모범 AI 분석 결과 1개
```

즉, 샘플 데이터는 문제와 정답이 붙어 있는 한 세트입니다.

## 3. 직무군 검수 기준

`jobFamily`는 프로필의 희망 직무, 기술, 경험과 맞아야 합니다.

| jobFamily | 적합한 예 |
| --- | --- |
| `DEVELOPMENT_DATA` | 백엔드 개발자, 데이터 분석가, QA, 인프라 엔지니어 |
| `SALES_MARKETING` | 콘텐츠 마케터, 영업관리, 광고 운영, CRM 담당자 |
| `DESIGN_CONTENT` | UX/UI 디자이너, 콘텐츠 디자이너, 영상 편집자 |
| `BUSINESS_OFFICE` | 서비스 기획자, 인사, 회계, 총무, 운영 |
| `HEALTHCARE_SERVICE` | 간호, 병원 행정, 상담, 고객 서비스 |
| `EDUCATION_PUBLIC` | 강사, 교육 운영, 행정, 공공기관 |
| `PRODUCTION_LOGISTICS` | 생산관리, 품질관리, 물류, 구매, 현장 운영 |
| `ENGINEERING_TECHNICAL` | 기계설계, 전기전자, 건설/토목, 설비, 화학, 바이오, 환경안전, 연구개발 |
| `GENERAL` | 희망 직무가 불명확하거나 여러 직무가 섞인 경우 |

잘못된 예:

```text
희망 직무: 콘텐츠 마케터
skills: SNS 운영, GA4, 카드뉴스
jobFamily: DEVELOPMENT_DATA
```

이 경우는 `SALES_MARKETING` 또는 데이터 내용에 따라 `DESIGN_CONTENT`가 더 적합합니다.

## 4. 점수 검수 기준

`rawScore`는 0에서 100 사이 정수입니다.
점수는 아래 기준으로 일관되게 작성합니다.

| 점수 범위 | 의미 |
| --- | --- |
| 90~100 | 매우 강함. 구체적 경험, 정량 성과, 직무 연결이 모두 명확함 |
| 75~89 | 좋음. 직무 연결과 경험은 좋지만 일부 근거가 부족함 |
| 60~74 | 보통. 기본 정보는 있으나 구체성이나 성과가 부족함 |
| 40~59 | 약함. 관련 경험이 적거나 설명이 추상적임 |
| 0~39 | 매우 부족. 해당 기준을 판단할 정보가 거의 없음 |

예를 들어 성과 수치가 전혀 없는데 `ACHIEVEMENT_EVIDENCE`가 95점이면 잘못된 데이터입니다.

## 5. 평가 기준별 검수 포인트

| 기준 | 확인 질문 |
| --- | --- |
| `GOAL_CLARITY` | 희망 직무, 산업, 목표가 명확한가 |
| `EXPERIENCE_SPECIFICITY` | 프로젝트/경력에서 역할, 사용 도구, 수행 업무가 구체적인가 |
| `ACHIEVEMENT_EVIDENCE` | 수치, 결과, 개선 전후, 사용자 반응 같은 근거가 있는가 |
| `JOB_SKILL_ALIGNMENT` | 보유 기술/역량이 희망 직무와 직접 연결되는가 |
| `DOCUMENT_CONSISTENCY` | 이력서, 자기소개, 프로젝트, 포트폴리오가 서로 같은 방향을 말하는가 |
| `IMPROVEMENT_READINESS` | 부족한 점을 사용자가 바로 보완할 수 있게 정리했는가 |

## 6. 추천 액션 검수 기준

추천 액션은 사용자가 바로 고칠 수 있는 문장이어야 합니다.

좋은 예:

```text
프로젝트 설명에 사용 기술, 담당 역할, 결과 수치를 각각 한 문장씩 추가하세요.
SNS 운영 경험에 도달률, 클릭률, 팔로워 증가 수치를 추가하세요.
UX 프로젝트에 문제 발견, 개선안, 개선 후 사용자 반응을 순서대로 정리하세요.
```

나쁜 예:

```text
더 노력하세요.
역량을 강화하세요.
경험을 더 쌓으세요.
```

나쁜 예가 문제인 이유는 사용자가 무엇을 수정해야 하는지 알 수 없기 때문입니다.

## 7. 직무군 분포 검수 기준

초기 30개 데이터셋을 만든다면 아래처럼 분산시키는 것이 좋습니다.

| 직무군 | 권장 개수 |
| --- | ---: |
| `DEVELOPMENT_DATA` | 5 |
| `SALES_MARKETING` | 5 |
| `DESIGN_CONTENT` | 4 |
| `BUSINESS_OFFICE` | 5 |
| `HEALTHCARE_SERVICE` | 3 |
| `EDUCATION_PUBLIC` | 4 |
| `PRODUCTION_LOGISTICS` | 3 |
| `ENGINEERING_TECHNICAL` | 3 |
| `GENERAL` | 1 |

개발 직무 데이터만 많으면 모델이 모든 프로필을 개발자 관점으로 평가할 수 있습니다.
CareerTuner는 다양한 직무의 취업 준비를 돕는 서비스이므로 비개발 직무 데이터가 반드시 필요합니다.

## 8. 개인정보 검수 기준

학습 데이터에는 실제 개인정보를 넣지 않습니다.

금지 항목:

- 실제 이름
- 실제 이메일
- 전화번호
- 주소
- 주민등록번호
- 실제 회사 내부 정보
- 실제 고객 정보

샘플에는 `example.com`, `한국대학교`, `에듀스타트`처럼 가상의 값을 사용합니다.

## 9. 자동 검증 실행

아래 명령으로 구조 검증과 일부 품질 경고를 확인합니다.

```bash
python docs/ai-training/validate_profile_ai_dataset.py docs/ai-training/profile_ai_seed_samples.jsonl
```

검증 스크립트가 확인하는 항목:

- JSONL 파싱 가능 여부
- assistant 응답 JSON 파싱 가능 여부
- 필수 출력 필드 존재 여부
- 6개 평가 기준 존재 여부
- 점수 범위 0~100 여부
- 직무군 분포
- 기준별 평균 점수
- 실제 이메일/전화번호/주민등록번호 의심 패턴
- 너무 추상적인 추천 문장
- 학습 준비용 최소 샘플 수 30개 충족 여부

## 10. 최종 학습 전 승인 기준

실제 학습에 넣기 전에는 아래 조건을 만족시키는 것이 좋습니다.

- 구조 검증 오류 0개
- 심각한 개인정보 경고 0개
- 최소 30개 이상
- 모든 주요 직무군 포함
- 특정 직무군이 전체의 40%를 넘지 않음
- 각 샘플의 추천 액션이 구체적임
- 사람이 최소 1회 이상 점수와 추천 문장을 검수함
