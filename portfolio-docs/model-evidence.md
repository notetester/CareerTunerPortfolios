# AI 모델 증거 매트릭스

CareerTuner는 "모델 파일이 있다"와 "제품이 그 모델을 기본으로 사용한다"를 같은 말로 취급하지 않습니다. 각 모델은 학습, 평가, 백엔드 연결, 배포 설정, 라이브 가용성을 따로 확인해야 합니다. 아래 상태는 공개 포트폴리오 소스 기준선 `d00a57fc`에서 코드와 재현 자료를 대조한 결과입니다.

| 영역 | 모델·기법 | 학습·평가 | 제품 연결 | 기본값·운영 경계 |
| --- | --- | --- | --- | --- |
| A 프로필 | 프로필 요약·스킬 추출용 파인튜닝 경로 | 모델 계약과 검증기가 존재 | `FineTunedProfileAiService`가 선택적으로 사용 | 기본 비활성. 미설정 시 Claude/OpenAI/규칙 기반으로 내려감 |
| B 공고 | PaddleOCR/PP-Structure + 공고 분석 LoRA | worker와 R1/R2 평가 자료가 분리됨 | OCR, 재추출, 직무·회사 분석 파이프라인에 연결 | strict 재추출은 사용자가 고른 provider를 유지하며 임의 교차 폴백하지 않음 |

## 공통 판정 기준

1. **학습 완료**: 데이터셋, 학습 파라미터, 체크포인트 또는 변환 산출물이 재현 경로에 있다.
2. **평가 완료**: fixture가 아니라 명시된 평가셋과 지표, 실패 사례가 남아 있다.
3. **연결 완료**: 서비스 코드가 실제 provider를 호출하고 구조화 결과를 검증한다.
4. **기본 활성**: 코드 기본 설정만으로 해당 provider가 선택된다.
5. **배포 활성**: 환경변수와 endpoint가 구성된 특정 배포에서 사용할 수 있다.
6. **라이브 확인**: 점검 시점에 provider health와 실제 응답 provenance가 확인됐다.

이 구분 덕분에 포트폴리오에서 학습 성과를 과장하지 않으면서도, "왜 전체 파인튜닝이 아니라 LoRA인가", "왜 RAG만 쓰지 않았는가", "외부 API가 끊기면 어떻게 되는가"를 코드 근거와 함께 설명할 수 있습니다.
