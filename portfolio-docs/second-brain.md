# Second Brain과 문서 근거

대규모 프로젝트의 문서를 매번 처음부터 읽지 않도록 CareerTuner는 정본 소스, 사람이 읽는 Wiki, 관계 그래프를 세 계층으로 나눕니다.

1. Graphify 보고서와 공개 graph에서 관련 허브·연결을 찾습니다.
2. Obsidian Wiki의 합성 문서에서 결정과 현재 해석을 읽습니다.
3. 구현을 바꿀 때만 main repo의 정본 문서와 runtime source로 내려갑니다.

비공개 vault에는 운영 메모와 raw 자료가 있을 수 있으므로 그대로 공개하지 않습니다. 포트폴리오에는 공개 가능성이 검토된 Wiki와 축약 graph만 `/Obsidian/`에 투영합니다. 장문 AI 실험 자료 역시 통째로 복사하지 않고, 이 설명서의 [모델 증거 매트릭스](./model-evidence.md)처럼 검증 가능한 사실과 한계를 재서술합니다.

이 구조의 목적은 문서 수를 늘리는 것이 아니라 다음 질문에 빠르게 답하는 것입니다.

- 어떤 기능이 어느 API·테이블·플랫폼과 연결되는가?
- 마지막 검증 이후 어떤 PR이 해당 사실을 바꿨는가?
- 구현 완료, 설정 필요, 외부 자격증명 대기, 미구현을 어떻게 구분하는가?
- 공개해도 되는 근거와 내부에 남겨야 하는 raw artifact의 경계는 어디인가?
