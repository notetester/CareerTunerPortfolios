# 보안 정책

## 지원 대상

CareerTuner는 현재 개발 단계의 프로젝트입니다. 정식 릴리즈 버전이 확정되기 전까지는 버전 번호가 아니라 브랜치를 기준으로 보안 수정 대상을 관리합니다.

| 대상 | 보안 수정 지원 여부 |
| --- | --- |
| `main` | 지원 |
| `dev` | 지원 |
| 기능 개발 브랜치 | 제한적 지원 |
| 배포되지 않는 실험/폐기 브랜치 | 미지원 |

보안 수정은 원칙적으로 `dev` 브랜치에 먼저 반영한 뒤, 검토 후 `main` 브랜치로 병합합니다.

## 취약점 신고 방법

CareerTuner에서 보안 취약점을 발견한 경우, 취약점의 세부 내용을 공개 이슈나 공개 PR에 바로 작성하지 마세요.

다음 방법 중 가능한 경로로 비공개 신고해 주세요.

1. GitHub의 Private vulnerability reporting 또는 Security Advisory 기능을 사용합니다.
2. 위 기능을 사용할 수 없는 경우, 저장소 소유자 또는 프로젝트 관리자에게 직접 전달합니다.

신고 시에는 가능한 범위에서 다음 내용을 포함해 주세요.

- 취약점에 대한 설명
- 재현 절차
- 영향을 받는 파일, API, 화면, 기능 또는 배포 환경
- 예상되는 보안 영향
- 관련 로그, 스크린샷, 요청/응답 예시
- 알고 있는 완화 방법 또는 수정 제안

유효한 보안 신고는 가능한 한 3영업일 이내에 1차 확인을 목표로 합니다.

## 보안 점검 범위

다음 항목은 보안상 중요하게 관리합니다.

- 회원가입, 로그인, 인증 및 인가
- OAuth 설정
- API 키, 토큰, 비밀값 관리
- 사용자 프로필, 이력서, 경력, 자격증 데이터
- 업로드된 공고문 및 문서 파일
- 관리자 기능
- AI 분석 로그 및 면접 데이터
- 데이터베이스 접근 권한
- GitHub Actions 및 배포 워크플로
- 데모 배포 환경 설정

## 범위 외 항목

다음 항목은 민감 정보 노출, 권한 상승, 인증 우회, 서비스 장애와 직접 연결되지 않는 한 일반적으로 보안 취약점 범위에서 제외합니다.

- 단순 UI 표시 오류
- 스타일 또는 레이아웃 문제
- 민감 정보가 포함되지 않은 콘솔 경고
- 개발용 목업 데이터 문제
- 실제 배포에 사용되지 않는 폐기 브랜치의 문제

## 공개 원칙

취약점이 확인된 경우, 수정이 완료되고 필요한 검토가 끝나기 전까지는 세부 내용을 공개하지 않습니다. 패치가 적용된 뒤 필요한 경우 영향 범위와 조치 내용을 정리해 공유합니다.
