# CareerTuner 보안 정책

CareerTuner는 채용공고, 사용자 경력 프로필, 자기소개서/문서 파일, 면접 연습 데이터, 관리자 운영 데이터를 함께 다루는 AI 취업 전략 플랫폼입니다. 보안 신고와 수정은 이 데이터가 외부에 노출되거나 잘못 사용될 가능성을 줄이는 데 초점을 둡니다.

이 문서는 저장소 운영자가 아직 확정하지 않은 릴리스 브랜치 이름, 장기 지원 버전, 보상 정책을 대신 정하지 않습니다. 브랜치와 PR 운영은 [AGENTS.md](AGENTS.md)와 [docs/BRANCHES.md](docs/BRANCHES.md)를 따르며, 보안 수정 대상은 저장소의 현재 기본 개발선과 실제 배포/데모에 연결된 활성 코드입니다.

## 신고 방법

보안 취약점의 세부 내용은 공개 이슈, 공개 PR, 공개 토론에 바로 올리지 마세요.

가능한 경우 GitHub의 비공개 취약점 신고 또는 Security Advisory 기능을 사용합니다. 그 경로를 사용할 수 없다면 저장소 소유자나 프로젝트 관리자에게 직접 전달합니다.

신고에는 가능한 범위에서 다음 정보를 포함해 주세요.

- 영향받는 기능, API, 화면, 워크플로, 파일 경로
- 재현 절차와 필요한 권한 수준
- 예상되는 영향: 개인정보 노출, 인증 우회, 권한 상승, 결제/크레딧 조작, 파일 접근, 서비스 중단 등
- 요청/응답 예시, 로그, 스크린샷, 테스트 계정 조건
- 이미 확인한 완화 방법이나 수정 제안

민감한 사용자 데이터, 실제 토큰, 실제 비밀번호, 원본 이력서/면접 영상/음성 파일은 신고 본문에 그대로 붙이지 마세요. 필요한 경우 마스킹한 샘플이나 최소 재현 데이터로 설명합니다.

## 우선 점검 영역

다음 영역은 CareerTuner에서 특히 보안 영향이 큰 부분입니다.

- 회원가입, 로그인, 세션, JWT, OAuth, 비밀번호 재설정
- 사용자/관리자 권한 분리와 관리자 API 접근 제어
- 사용자 프로필, 경력, 자격증, 자기소개서, 지원 건 데이터
- 채용공고 업로드, 문서 추출, OCR, 파일 저장/다운로드
- 면접 음성/영상, 비언어 분석, 면접 리포트, 전사/평가 데이터
- AI 프롬프트, 모델 응답, 분석 로그, RAG/학습/평가 데이터셋
- 결제, 구독, 크레딧, AI 사용량 차감 로직
- MyBatis 매퍼, DB 권한, 마이그레이션, 운영 SQL
- GitHub Actions, 배포 토큰, 데모 배포, 모바일 빌드, 4090 작업 트리거
- `.env`, API 키, VAPID/FCM/APNs, SMTP, OpenAI/Ollama/외부 모델 접속값

## 범위 밖 또는 낮은 우선순위

다음 항목은 민감 정보 노출, 권한 문제, 무단 데이터 변경, 서비스 장애와 직접 연결되지 않는 한 보안 취약점으로 처리하지 않을 수 있습니다.

- 단순 오타, 스타일, 반응형 레이아웃 문제
- 민감 정보가 없는 콘솔 경고나 개발용 로그
- 실제 배포/데모와 연결되지 않은 폐기 실험 코드
- 공개 가능한 목업 데이터의 표시 오류
- 악용 가능성이 확인되지 않은 일반적인 의존성 최신화 요청

의존성 보안 업데이트는 `.github/dependabot.yml`에서 프런트엔드, 백엔드, Python ML/OCR 워커, Docker 이미지, GitHub Actions를 나누어 관리합니다. 실제 악용 가능한 취약점이나 공개 CVE가 포함된 업데이트는 일반 버전 업데이트보다 우선 검토합니다.

## 수정과 공개

취약점이 확인되면 필요한 범위에서 비공개 브랜치, 제한된 PR, 또는 Security Advisory로 수정 내용을 관리합니다. 패치가 준비되기 전까지 재현 코드와 영향 범위 세부 사항은 공개하지 않습니다.

비밀값이 노출된 경우에는 코드 수정만으로 끝내지 않고 해당 키를 폐기하거나 교체합니다. 이미 커밋된 비밀값은 단순 삭제 후 재사용하지 않습니다.

현재 이 저장소에는 별도의 보안 보상 프로그램이나 고정 응답 시간이 정의되어 있지 않습니다. 관리자는 신고의 재현 가능성, 영향 범위, 배포 여부를 기준으로 우선순위를 정합니다.
